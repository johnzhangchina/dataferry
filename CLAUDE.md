# NianHe（粘合）— 轻量级 API 胶水平台

## 项目概述

解决企业内部及企业间系统不互通的痛点：A 系统有 Webhook 推送，B 系统有 OpenAPI，双方项目组都不愿意改。NianHe 做中间的转换和转发层，两边零侵入。

## 产品定位

**"不改代码，粘合任意两个系统"** — 通用的、不绑定具体平台的 API 胶水工具。

不做预置连接器（不和 Zapier/腾讯轻联竞争），用户自己配置两端，我们只做中间的转换引擎。

## 起源

源于真实的工作场景：两个子公司系统数据不互通，项目组都不愿意在自己平台上给对方做私有化对接。

## 核心功能

```
任意 Webhook 进来 → 可视化字段映射/转换 → 调用任意 API 出去
```

### MVP 三件事

1. **接收** — 给用户一个专属 URL，接收任意 JSON 推送
2. **映射** — 可视化界面配置字段映射，支持：
   - 直接映射：`A.name → B.username`
   - 默认值：`"子公司A" → B.source`
   - （后续）拼接：`A.first + A.last → B.fullname`
   - （后续）简单转换：`A.price * 100 → B.amount_cent`
3. **调用** — 按用户配置的目标 API 格式发出请求

### AI 辅助（核心差异化）

用户粘贴 A 的 Webhook payload 示例 + B 的 API 文档，AI 自动生成映射关系，用户确认/微调即可。

## 目标用户

有一定技术认知、但不想写代码的人：
- 企业 IT 管理员 / 实施顾问
- 知道 Webhook/API 概念但不写代码的技术人员
- 需要对接内部系统的项目经理

## 差异化

| 对比 | Zapier/腾讯轻联 | NianHe |
|------|----------------|--------|
| 模式 | 预置连接器，绑定具体平台 | 通用引擎，不绑定任何平台 |
| 覆盖 | 大厂之间的连接 | 小厂/内部系统/冷门系统之间 |
| 核心 | 连接器数量 | 字段映射与转换能力 |
| 开源 | 否 | 是 |

## 商业模式

- **开源（免费）**：转换引擎、可视化映射、所有核心功能，用户自行部署
- **云托管版（收费）**：免部署、免运维、高可用

## 起步策略

1. **先解决自己的问题** — 用 NianHe 打通两个子公司的系统对接
2. **自己用着好用后开源** — 作为信任背书和获客手段
3. **SaaS 托管版盈利** — 面向不愿自部署的用户
4. **内容获客** — 围绕具体场景写文章/录视频

## 市场定位

不做大厂之间的胶水（已有成熟方案），专注：
- 企业内部子系统之间的对接
- 小众/垂直行业 SaaS 之间的对接
- 自建系统/私有化部署系统的对接

这些是大厂连接器覆盖不到的长尾场景。

## 合规备忘

国内运营需要：
- ICP 域名备案（1-3 周）
- 公安网备案
- 企业主体
- 收费运营需 ICP 经营许可证
- 涉及数据交换可能需要 EDI 许可证（B21）

早期部署在境外可暂时跳过。
